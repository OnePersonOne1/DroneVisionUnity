# 조작법 (DroneVisionUnity)

Unity 플레이 모드에서의 키보드 조작 및 주요 기능 정리.
입력은 **New Input System** 사용(프로젝트 `activeInputHandler: 2`).

> **두 종류의 드론**이 공존한다.
> - **Cube** = *카메라 제어용*. WASD/↑↓/←→/R-F로 시점/카메라 위치 조작 + 검출 마커 표시·HUD 의 기준점.
> - **`drone_sim_obj_n`** (n = 1, 2, 3, …) = *6-DOF 동역학 시뮬레이션 드론*. 자체 EOM + 제어기로 비행, C2(미니맵·전략뷰) 명령 수신.
>
> 키는 **별도 그룹**으로 매핑돼 충돌 없음 (드론 시뮬은 주로 마우스 + Function 키).

## 키 요약 — Cube (카메라 / HUD / 추론)

| 키 | 기능 | 비고 |
|---|---|---|
| **W / A / S / D** | 수평 이동 (전 / 좌 / 후 / 우) | `CubeDroneController` |
| **↑ / ↓** | 상승 / 하강 (고도) | 화살표 위·아래 |
| **← / →** | 방향 회전 — Yaw 좌 / 우 | 화살표 좌·우 |
| **R / F** | 기수(Pitch) 올림 / 내림 | 자세 표현용 |
| **LMB 드래그** (좌클릭) | Cube yaw + pitch 마우스 룩 — **1인칭/3인칭 양쪽 공통** | `CubeDroneController.enableMouseLook` |
| **V** | (멀티 디스플레이 모드면 무력화) FP 라벨/크로스헤어 토글만 | `FirstPersonView`. `MultiDisplayCoordinator` 가 카메라 enabled 강제 유지. |
| **M** | FP 모드 전환: CubeFree ↔ SensorFollow | 〃 |

| **J** | **Identity-quat 비교 토글** (회전 미적용 시 마커 위치 시각 비교) | `ProjectionReplay`, `ProjectionUdpReceiver` |
| **B** | **중앙 픽셀 건물 정보 조회 + 선택 추가** (색상 카드 + 건물 색 오버레이) | `BuildingInfoProbe` → `BuildingInfoService` → `ARInfoPanel`/`BuildingFootprintHighlighter` |
| **N** | 건물 조회 dwell-auto 모드 토글 | `BuildingInfoProbe` |

| **X** | **건물 정보 선택 전체 해제** (모든 카드 + 색 오버레이 제거) | `BuildingInfoService` |
| **`\`** | **건물 정보 조회 (Display 4 sensor camera 중앙)** | `BuildingInfoProbe.sensorProbeKey` |

## 키 요약 — 모의 화재 + 상황 판단 (Situation Briefing)

`/assess` 백엔드와 `FireSim` / `SimClock` / `SituationAssessClient` / `BriefingPanel` 4 컴포넌트 묶음.
모의 화재 또는 RF-DETR 검출 화재(`fire_region`/`smoke_region`) 가 1건이라도 있으면 즉시 평가.

| 키 / 입력 | 기능 | 위치 |
|---|---|---|
| **`;`** | FP 카메라 중앙 raycast 점에 **fire 마커** 주입 (`fire_region`, 기본 영구 — `FireSim.fireMarkerLifetime`) | `FireSim.fireKey` |
| **`'`** | 동일 위치에 **smoke 마커** 주입 (`smoke_region`, 기본 영구) | `FireSim.smokeKey` |
| **Delete** | 모의 화재 일괄 제거 (검출 화재는 영향 X) | `FireSim.clearKey` |
| **Z** | **SimClock 패널** 토글 — 시간대 프리셋(실시간/08:00/12:00/18:00/02:00) + 수동 시:분 입력 | `SimClockUI.toggleKey` |
| **F12** / **Shift+F12** | **Briefing 패널** 토글 / 기본 크기 리셋 | `BriefingPanel.toggleKey` |
| **F11** | 수동 즉시 `/assess` 호출 (개발용) | `SituationAssessClient.manualKey` |
| **`[`** (LeftBracket) | **FP(Display 1) ↔ Sensor(Display 4) 카메라 교환** — 3 모니터 환경에서 Display 4 가 안 보일 때 1번 자리에 끼워넣기. 같은 키 다시 누르면 원위치. | `MultiDisplayCoordinator.swapFpSensorKey` |

### PX4 SITL 드론 (`drone_sitl_N`)

`docker-compose.sitl.yml` 의 px4_sitl + px4_bridge 가 떠 있을 때 자동 등록되어 미니맵·전략 뷰에 **주황** 마커로 표시. 기존 시뮬 드론과 동시 표시·각각 제어. **이동 명령(RMB SetWaypoint / Ctrl+드래그 SetPath / Shift+RMB Queue) 은 기존 키 그대로** — 선택군에 SITL 드론이 포함되면 자동 라우팅. 아래는 SITL 전용 직접 제어 키.

| 키 | 기능 | 비고 |
|---|---|---|
| **Numpad7** | ARM | `SitlControlInput.armKey`. 노트북에서 Numpad 없으면 인스펙터로 변경. |
| **Numpad8** | TAKEOFF (기본 5m AGL) | `takeoffAltMeters` 인스펙터 조정. |
| **Numpad9** | LAND | 〃 |
| **Numpad6** | RTL (Return-to-Launch, 홈으로 복귀) | 〃 |
| **F9** | SITL 드론 추가 spawn | `SitlDroneSpawner.spawnKey` — 현재 1대 전제 (다중 미지원, 아래 참고). |

대상: `StrategicCommandInput.selectedDroneIds` 에 명시 선택된 드론 중 `MavlinkFlightModel` 인 것. 선택 비어있으면 전체 SITL 드론. 시뮬 드론(`HighFidelityFlightModel`) 은 영향 X.

#### SITL 다중 드론 — 현재 미지원

| 항목 | 상태 |
|---|---|
| 시뮬 드론 (`drone_sim_obj_N`) | **N 대 동시 가능** (DroneSpawner 가 시리얼 부여, 각자 자체 EOM) |
| SITL 드론 (`drone_sitl_N`) | **1대 전제** — MavlinkFlightModel 이 udp:9871/9872 단독 bind, 다중 spawn 시 포트 충돌 |
| 혼합 | **시뮬 N 대 + SITL 1 대** 까지 OK |

다중 SITL 활성화하려면 PX4 SITL 인스턴스를 별 포트(14541, 14542, …) 로 분리 + 브리지가 N 인스턴스 다중화 + Unity 측 9871 단일 socket 을 드론 id 라우터로 재설계 필요. 별도 후속 작업.

#### PID 튜닝 / 응답 분석 — QGroundControl + 자동화 스크립트

산업 표준 GCS 인 **QGroundControl** 을 호스트에서 실행해 PX4 SITL 에 직접 연결:

| 단계 | 동작 |
|---|---|
| 1 | 호스트에서 `./QGroundControl-x86_64.AppImage` 실행 |
| 2 | 좌상단 `Q` → **Application Settings** → **Comm Links** → **Add** (UDP, Listening Port `14540`, Server `127.0.0.1:14540` Add → OK) → **Connect** |
| 3 | default 화면 = **Fly View** (인천 지도 + attitude indicator). Ready to Fly 표시 시 takeoff slider 슬라이드 |
| 4 | **Vehicle Configuration** → **PID Tuning** — Rate/Attitude/Velocity/Position 4 cascaded 단계, 슬라이더로 실시간 게인 변경 |
| 5 | **Autotune** 탭 — 호버 중 trigger. SITL 에선 mavlink packet loss 로 fail 흔함, manual sweep 으로 우회 권장 |

> ⚠ **`px4_bridge` 와 QGC 가 둘 다 14540 점유 불가**. QGC 사용 중에는 `docker stop px4_bridge` 로 잠시 정지 (Unity SITL 마커 일시 stale), 끝나면 `docker start px4_bridge` 로 복귀.

자동화 스크립트 (포트폴리오 산출물):

| 스크립트 | 출력 |
|---|---|
| `python3 Python/sitl/step_response.py --axis x --amplitude 10` | `step_responses/*.csv` + `*.png` — 단일 step 응답 + 메트릭 (overshoot/rise/settling/ss_error) |
| `python3 Python/sitl/pid_sweep.py --param MC_ROLLRATE_P --values 0.10,0.15,0.20,0.25` | `pid_sweep/*.png` (overlay) + `_metrics.csv` — 게인별 비교 |
| https://logs.px4.io 업로드 (ulog) | Flight Review 자동 분석 (PID tune health 등급) |

QGC 동시 실행 시 스크립트의 `--conn udpout://127.0.0.1:14580` 사용 (PX4 instance 1 의 inbound).

**자동 트리거**: 화재 추가/삭제 시 즉시, SimClock 변경 시 0.6s 디바운스 후 자동 평가. RF-DETR 검출의 `fire_region`/`smoke_region` 마커도 `SituationAssessClient.includeDetectionFires` (기본 ON) 로 자동 합류 — 검출만 들어와도 평가됨.

**브리핑 패널 표시**:
- 위험도 뱃지 (high=빨강 / mid=주황 / low=초록) + 룰 기반 reason.
- 시간대 라벨 (출근/주간/점심/퇴근/심야), 화재·연기 건수, 확산 판정.
- LLM 또는 template 한국어 브리핑 (Ollama hang/미가동 시 자동 폴백, `INFO_LLM_TIMEOUT` 기본 15s).
- **가까운 119 안전센터 top-5** (CSV 로컬, 인천광역시 57개) — 거리·총 보유 차량·차종 목록.
- **권장 차량** (룰 산출) — 펌프차·물탱크차·구급차 기본 + 고층(≥10층 or 30m+) → 사다리차·굴절차 + 산업/위험물 → 화학차·고성능화학차·무인방수차 + 교육시설 → 생활안전차 + 다중화재 → 펌프차 +1대. top-5 inventory 와 비교해 `✓ 충당` / `⚠ 부족` 표시.
- 현장 특성 플래그 (고층·산업/위험물·교육시설·인원밀집·다중화재·핫존).
- provenance 푸터 — 근거 건물 수, 화재 ID 수, 반경, briefing_source(llm/template).

**룰 vs LLM**: risk·spreading·occupancy·차량 권장은 모두 **결정론 룰**(LLM 미관여). LLM 은 정형 fact 를 한국어 3~5문장으로 합성만. Ollama 미가동/hang 이어도 동일 fact 로부터 template briefing 즉시 반환.

## 멀티 디스플레이 분담

| 디스플레이 | 카메라 / 내용 | UI |
|---|---|---|
| **Display 1** (idx 0) | 1인칭 (FP Camera) | GPS HUD, AR 카드, Flight Console, 미니맵, 기타 모든 UI |
| **Display 2** (idx 1) | 3인칭 (Main Camera / DroneFollowCamera) | 없음 (순수 영상) |
| **Display 3** (idx 2) | 전략 보기 — 탑다운 ortho 전체화면 | 드론·타겟·큐 마커 + 박스/경로 시각화 + AGL 라벨 |
| **Display 4** (idx 3) | **센서 시점 뷰** (`SensorViewMode` 가 전용 카메라 자동 생성) | 상단 모드 배지 (FIXED/FREE, flip 상태) |

`MultiDisplayCoordinator` 가 Display 1/2/3 의 Activate + targetDisplay 강제 + V 토글 부작용(카메라 enabled / CubeDroneController.enabled) 매 프레임 보정. **`SensorViewMode` 는 자기 카메라를 Display 4 로 직접 활성화** — 단일 모니터 Editor 면 Game view 탭에서 `Display 4` 선택해 확인.

## 센서 시점 뷰 (Display 4)

`SensorViewMode` 가 추론 패킷(UDP) 또는 CSV(Replay) 의 `cam_lat/lng/alt` + 방향에 따라 **전용 카메라**를 센서 위치에 고정하고, 자세는 토글로 선택.

| 키 | 기능 |
|---|---|
| **U** | 센서 시점 모드 on/off (Display 4 카메라 enabled). **기본 ON** — Play 시작과 동시에 켜짐. |
| **F8** | 자세 모드 — **FIXED**(센서 reported orientation) ↔ **FREE**(LMB 드래그 마우스 룩) |
| **F7** | **roll 180° 뒤집기 토글** — 센서가 거꾸로 찍힌 영상일 때 화면 보정 (FIXED 에만 적용, 기본 ON) |

- **FIXED**: `Quaternion.LookRotation(fwd, up)` + 옵션 roll 180°.
- **FREE**: 위치는 센서 GPS 에 고정, 자세는 **LMB 드래그**로 yaw/pitch 자유 조작 (다른 시점과 동일, 항상 upright — RMB 는 strategic SetWaypoint 용으로 비워둠).
- Display 1 의 FP 카메라와 **완전 독립** — FP 뷰는 영향 받지 않음.

## 키 요약 — `drone_sim_obj_n` (6-DOF 동역학 시뮬레이션)

Cube 와 무관한 별도 드론 객체. 런타임에 spawn, 명령은 마우스 클릭 + 정적 API.

| 키 / 입력 | 기능 | 위치 |
|---|---|---|
| **`** (백쿼트) | 드론 1대 추가 spawn (`drone_sim_obj_n`, n = 1, 2, 3, …) | 어디서나 |
| **F10** | 비행 모드 토글 — 전체 드론 (HighFidelity ↔ Arcade) | 어디서나 |
| **LMB 클릭 (드론 마커)** | 단일 드론 선택 | 어느 디스플레이든 |
| **Shift + LMB 클릭 (드론 마커)** | 선택군에 추가/제거 | 어느 디스플레이든 |
| **LMB 드래그 (빈 공간, Display 3 또는 미니맵)** | 박스 셀렉트 → 박스 안 드론 다중 선택 | 전략 뷰 / 미니맵 |
| **LMB 단일 클릭 (빈 공간, Display 3 또는 미니맵)** | 선택 해제 → **전체 드론 대상** 으로 복귀 | 전략 뷰 / 미니맵 |
| **Ctrl + LMB 드래그 (Display 3 또는 미니맵)** | 마우스 궤적 샘플링 → 선택군에 `SetPath` | 전략 뷰 / 미니맵 |
| **RMB 클릭** | 선택군에 `SetWaypoint` (해당 디스플레이 카메라 ray → ground hit; 미니맵은 클릭 XZ + 드론 현 고도) | 어느 디스플레이든 |
| **Shift + RMB** | `QueueWaypoint` (큐에 추가) | 〃 |
| **ESC** | **선택 해제** → 전체 드론 대상 (어느 디스플레이에서나) | `StrategicCommandInput.clearSelectionKey` |
| **PgUp / PgDn** | 명령 고도(AGL) ±5 m (PgDn 은 0 하한) | `StrategicCommandInput.commandAltitudeAGLMeters` |
| **End** | 명령 고도 = 0 (= 클릭한 지면 그대로) | 〃 |
| **F1 / F2 / F3 / F4 / F5** | 명령 고도 프리셋 = 10 / 30 / 50 / 100 / 200 m (숫자 1–9 는 컨트롤 그룹용으로 비움) | 〃 |
| **Ctrl + 1~9** (또는 **Alt + 1~9**) | **컨트롤 그룹 저장** — 현재 선택을 그룹 N 에 스냅샷 (RTS 부대지정). 선택이 비어 있으면 전체 드론. Editor 의 Ctrl+digit 가 Scene/Game/Inspector 등으로 가로채면 Alt+digit 사용. | 〃 |
| **1~9** (단독) | **컨트롤 그룹 호출** — 그룹 N 을 선택군으로 복원. 미저장 그룹은 무시. | 〃 |
| **↑↓←→ (Display 3)** | 전략 카메라 팬 | 전략 뷰 |
| **MMB 드래그** | 전략 카메라 팬 | 어디서나 |
| **마우스 휠** | 전략 카메라 줌 | 어디서나 |
| **Home** (또는 **Backspace**) | 전략 카메라 → Cube 위로 재중심 (줌 유지). 노트북에서 Home 이 Fn 레이어면 Backspace 사용. | 어디서나 |
| **F9** | **HUD 레이아웃 튜너 창 열기/닫기** (debug HUD/범례/스케일바 크기·폰트·알파 슬라이더) | `StrategicViewHud` |
| **Space** | **전체 드론 정지** — 모든 드론의 wp 큐 비우고 현 자리에서 hover (RTS 의 S=Stop) | `StrategicCommandInput.stopAllKey` |

**명령 고도(AGL) 의미**: `commandAltitudeAGLMeters > 0` 이면 `SetWaypoint/SetPath` 가 클릭한 지면 위 해당 AGL 미터로 wp 를 띄움. `0` 이면 드론 현재 고도 그대로 유지.

**선택 시맨틱**: `selectedDroneIds` 가 빈 리스트면 "전체 대상". 명령(RMB, SetPath) 이 모든 등록 드론에 일괄 전달. 명시 선택 시 그 드론들만.

**Spawn 위치**: `DroneSpawner.spawnAnchor`(비우면 자동으로 `Cube`) 의 **forward × 5m** (Unity 단위 = real m × `unityUnitsPerMeter`).
**외관**: 두 대각 막대 + 4 모터 구. 전우(m0) 모터 빨강 (전방 표시).
**Spawn 모드** (`DroneSpawner.Initial Mode`): `FreeFall / HoverInPlace / AttitudeHold / Navigate`. Navigate 가 기본 비행 — Position+Attitude+Rate cascade PID 가 wp 추종.

## 전략 보기 (Display 3) HUD 마커

| 마커 | 라벨 |
|---|---|
| 회색 큰 원 | `ME` (Cube 위치) |
| 시안 원 | 드론 — 라벨 = `{N}\nAGL: {m} m` (드론 번호 + 지상 고도, N = 1, 2, 3, …) |
| 노랑 원 | 선택된 드론 |
| 녹색 원 | 현재 타겟 waypoint — 라벨 = 어느 드론들이 향하는지: `1`, `1,2`, `all` |
| 작은 녹색 점 | 큐 잔여 waypoint (동일 라벨 규칙) |
| 반투명 녹색 사각형 | LMB 드래그 중 박스 셀렉트 영역 |

같은 위치(pixel bin)의 다중 드론 타겟은 **자동 그룹핑**되어 한 마커로 표시 (라벨 합성).

  - **지면 원반** (`showGroundDiscs`, 반경 6 m, alpha 0.55) — wp XZ 에서 아래로 raycast(`groundMask`) 한 지점에 평평한 디스크. 색은 원기둥과 다르게 **선택 노랑 / 비선택 주황** (RTS 스타일 도착 마커). 인스펙터 `discRadiusMeters`, `discAlpha`, `discLiftMeters`, `discDefaultColor` / `discSelectedColor` 조정.

## HUD 창 공통 (`DraggableHud`)

모든 UGUI HUD 패널을 **드래그·접기·키 토글·리사이즈** 가능한 작은 윈도우로 만드는 래퍼.

- **헤더 드래그** = 위치 이동 (`lockPosition=true` 면 비활성, 예: 미니맵).
- **`−`** 헤더 버튼 = 접기 / 펴기.
- **`×`** 헤더 버튼 (toggleKey 가 있을 때만) = 숨기기.
- **toggleKey** (인스펙터 설정) = 보이기/숨기기 토글 — 패널 비활성 상태에서도 작동.
- **◢ 그립** (resizable=true 일 때) 우상단 모서리 드래그 = 크기 조절 (`minSize` / `maxSize` 클램프).
- **resetSizeKey** (옵션, modifier 가능): 패널 크기를 Start 시점 초기 크기로 복귀. 미니맵 기본 = **Shift+M**.

## 1. 이동

- **W A S D**: 수평면 이동. 동시에 누르면 대각선(속도 보정됨).
- **↑ / ↓**: 고도 상승 / 하강 (Y축).
- 속도는 `CubeDroneController` 인스펙터 `moveSpeed`, `verticalSpeed`로 조절.

## 2. 방향 회전

- **← / →**: Yaw(좌우 선회). 이동 방향도 함께 바뀜.
- **R / F**: Pitch(기수 올림/내림). `minPitch`~`maxPitch`(기본 ±45°)로 제한, 자세 표현용이며 이동 방향에는 영향 없음.
- 회전 속도: 인스펙터 `yawSpeed`, `pitchSpeed`.

## 3. 순간이동 (텔레포트 / GPS 오버라이드)

지정한 위도·경도로 "순간이동"하여, **수신되는 GPS를 무시하고 그 좌표를 추론
입력으로 사용**하는 기능.

**설정 방법**
1. 씬에 빈 GameObject 만들고 `GpsTeleport` 컴포넌트 추가.
2. 인스펙터에서:
   - `Latitude` / `Longitude` / `Altitude` — 이동할 목표 좌표 입력.
   - `Drone` (선택) — 시점/드론 오브젝트(예: Cube) 연결 시 그 위치로 화면도 이동.
   - `Receiver` (비우면 자동 검색) — 좌표 변환에 `ProjectionUdpReceiver` 사용.
3. `ProjectionUdpReceiver`(및 `ProjectionReplay`)의 `Teleport` 필드는 비워두면
   자동으로 이 컴포넌트를 찾음.

**사용**
- **T** 키 또는 인스펙터 `Override Enabled` 체크 → 오버라이드 ON.
- ON 동안: 모든 검출이 입력 GPS 대신 지정 좌표 기준으로 맵에 표시됨.
  `Drone` 지정 시 해당 위치로 시점도 텔레포트.
- 실행 중 `Latitude/Longitude/Altitude`를 바꾸면 즉시 반영.
- 다시 **T** → OFF, 정상적으로 수신 GPS 사용.
- 토글 키 변경: `GpsTeleport.toggleKey` (기본 `T`).

## 4. 추론 on/off

- **I** 키 또는 `ProjectionUdpReceiver` 인스펙터 `Inference Enabled` 체크.
- OFF: 수신 패킷은 계속 받지만(백로그 방지용으로 버림) **마커를 생성하지 않음**.
- ON: 검출을 맵에 표시 재개.
- 토글 키 변경: `ProjectionUdpReceiver.inferenceToggleKey` (기본 `I`).

## 5. Debug 시각화 (광선 + GPS 원점)

- **O** 키 또는 인스펙터 `Show Debug Visuals` 체크.
- ON: 검출별로
  - **GPS 원점 구**(카메라 위치를 월드로 환산한 지점)
  - **광선 LineRenderer**(원점→방향, 클래스 색상)
  를 **런타임 Game 뷰**에 표시. 광선이 어디로 쏘이는지 직접 확인 가능.
- 인스펙터 노브:
  - `Origin Sphere Scale` — 원점 구 지름(기본 5).
  - `Ray Line Width` — 광선 두께(기본 0.5).
  - `Ray Draw Distance` — 광선 길이(기본 1000).
  - `Debug Visual Lifetime` — 0=영원, >0=초 단위로 자동 소멸. (`Sync Ray Lifetime With Marker` 가 켜져 있으면 무시되고 marker 와 동일 수명.)
  - `Sync Ray Lifetime With Marker` — 켜면 광선/원점 구가 marker 와 같은 시간에 자동 소멸(기본 ON). marker 가 영구(=0)면 광선도 영구. 끄면 위의 `Debug Visual Lifetime` 사용.
- **C** 키: 화면에 남아 있는 DebugOrigin/DebugRay 를 **즉시 일괄 삭제**
  (`Debug Visual Lifetime = 0`(영원)으로 누적된 것들 정리할 때 유용).
  토글 키 변경: `ProjectionUdpReceiver.clearDebugKey` (기본 `C`).
- `Draw Debug Rays`(Scene 뷰 전용 `Debug.DrawRay`)는 에디터 디버깅용 별도 옵션.

## 마커 표시 강화

- `Marker Prefab` 을 비워두면 **세로 막대(emissive Capsule)** 마커가 런타임에
  자동 생성됨(피벗=바닥, 클래스 색상으로 발광). 멀리서도 잘 보임.
- 크기 조절: 인스펙터 `Marker Scale`(기본 3 → 폭 3m × 높이 ~30m).
- 사용자 정의: 직접 만든 프리팹을 `Marker Prefab` 슬롯에 드래그. 자식 Renderer
  들이 모두 클래스 색상으로 자동 tint 됨.
- 수명: `Marker Lifetime` 0=영원(권장 Replay), >0=초 단위 자동 소멸.

## 드론 시뮬레이션 (`drone_sim_obj_n`)

PhysX arcade 가 아닌 자체 EOM + RK4 적분 + cascaded 제어기로 비행하는 **독립**
6-DOF 드론. Cube(카메라) 와 별개 GameObject 로 런타임에 spawn.

- 좌표계: **ENU(Core) ↔ Unity world** 변환은 어댑터 경계 한 곳(`FrameConversion`).
- 적분: 고정 dt RK4 (`FlightConfig.fixedStep` 기본 0.0025s = 400Hz).
- PhysX 미사용: `Rigidbody.isKinematic=true` 강제, Transform 은 EOM 결과로 매 tick 덮어쓰기.
- 지면: `Physics.Raycast` 로 명시적 clamp (`enforceGround=true`).

자세한 검증·씬 설정 절차와 콜라이더 부착 방법은 **README.md 의 "드론 시뮬레이션
(Flight + C2)"** 섹션 참고.
